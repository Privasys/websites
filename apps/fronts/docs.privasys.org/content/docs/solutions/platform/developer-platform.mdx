The Privasys Developer Platform at [developer.privasys.org](https://developer.privasys.org) provides a self-service interface for building, deploying, and managing confidential applications. Authenticate with GitHub via OIDC, choose a deployment target, upload your code, and your application runs inside hardware-protected infrastructure with attestation built in.

## Deployment Targets

### WASM Modules

Upload a pre-compiled `.cwasm` file or link a GitHub repository. The platform compiles your code via GitHub Actions and deploys it inside **Enclave OS Mini** (Intel SGX).

- Smallest possible Trusted Computing Base (~4 MB enclave binary)
- Application code runs as a WebAssembly module inside the enclave
- Ideal for security-critical workloads: secrets management, cryptographic operations, key custody

### Containers

Provide a container image reference. The platform deploys it inside **Enclave OS Virtual** with hardware-encrypted memory and full attestation.

- Standard Linux containers, standard tooling
- No code changes required
- Ideal for existing applications, AI/ML inference, data processing pipelines

## Authentication

Developers sign in with their **GitHub identity** via OIDC. The login page automatically redirects to the identity provider — no extra clicks required. Returning users with an existing session skip the login form entirely thanks to the `select_account` OIDC prompt. After the initial sign-in, developers can optionally activate a **Passkey** for passwordless access.

## Workflow

| Step | Action |
|------|--------|
| **1** | Sign in with GitHub |
| **2** | Create an application and choose a deployment target (WASM or Container) |
| **3** | Upload a `.cwasm` file, push a container image, or link a GitHub repository |
| **4** | The platform deploys your code inside a hardware-protected enclave |
| **5** | Clients verify the service through a standard RA-TLS connection |

## Attestation

Every deployment is automatically attested. The platform generates RA-TLS certificates that carry hardware attestation evidence. Clients connecting to your service verify the attestation during a standard TLS handshake, with no custom protocol and no SDK.

## Secrets Management

Applications deployed on the platform can use **Enclave Vaults** for hardware-protected key storage and secrets management. Keys are sealed to the enclave's identity and never exist in cleartext outside the trust boundary. See [Enclave Vaults](/docs/solutions/enclave-vaults/overview) for details.
