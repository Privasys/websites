The Privasys Platform is the complete infrastructure for building, deploying, and discovering confidential applications. It consists of two core components:

- **[Developer Platform](/docs/solutions/platform/developer-platform)** -- A self-service interface at [developer.privasys.org](https://developer.privasys.org) for deploying WASM modules and containers on hardware-protected infrastructure.
- **[App Store](/docs/solutions/platform/app-store)** -- A country-scoped catalogue at [privasys.org/uk/apps](https://privasys.org/uk/apps) where verified confidential applications are listed and discoverable.

Both components are built on top of Privasys's confidential computing stack: Enclave OS, hardware attestation, and RA-TLS verification.

## How the Platform Works

The platform runs applications inside **Confidential Virtual Machines** (Intel TDX, AMD SEV-SNP) or **SGX enclaves** (Intel SGX), and exposes them over **attested HTTPS**. An attested reverse proxy terminates TLS inside the trusted execution environment:

1. Generates TLS keys **inside the TEE**
2. Embeds a **hardware attestation quote** in the X.509 certificate
3. Terminates TLS and proxies traffic to your backend application
4. Clients see a standard HTTPS connection with attestation evidence in the certificate

## When to Use the Platform

| Scenario | Why? |
|----------|------|
| **Existing web applications** | No code changes. Add an attested reverse proxy, enable attested connections. |
| **Container-based deployments** | Your Docker/Kubernetes workloads run as-is inside the Confidential VM. |
| **Quick adoption** | Standard Linux, standard toolchains, standard deployment. No SGX SDK. |
| **TDX/SEV cloud instances** | Available on Google Cloud, Azure, and OVH Cloud. |

## Compared to Enclave OS

For maximum security with the smallest Trusted Computing Base, use [Enclave OS (Mini)](/docs/solutions/enclave-os/enclave-os-mini/architecture) (Intel SGX). For container workloads in Confidential VMs, use [Enclave OS (Virtual)](/docs/solutions/enclave-os/enclave-os-virtual/architecture).

| | **Platform** (Confidential VMs) | **Enclave OS** (SGX) |
|--|---------------------------|----------------|
| **TEE** | Intel TDX / AMD SEV-SNP | Intel SGX |
| **TCB** | Full VM (kernel + all apps) | ~4 MB enclave binary |
| **Code changes** | None | Application built as WASM module |
| **Deployment** | Standard Linux + attested proxy | Custom Rust build + SGX SDK |
| **Best for** | Existing applications, quick adoption | Security-critical, minimal TCB |

See [Trust Model](/docs/technology/confidential-computing/trust-model) for a deeper comparison.

## Platform Components

| Component | Description |
|-----------|-------------|
| [Developer Platform](/docs/solutions/platform/developer-platform) | Self-service interface for deploying WASM modules and containers on confidential infrastructure. |
| [App Store](/docs/solutions/platform/app-store) | Verified catalogue of confidential applications, backed by hardware attestation. |
| [Verification Libraries](/docs/solutions/platform/verification-libraries) | Libraries in Python, Go, Rust, TypeScript, and C# for verifying attested connections with a single function call. |
| [Attestation Server](/docs/technology/attestation/attestation-server) | A standalone service that independently verifies hardware attestation quotes from Intel and AMD platforms. |
